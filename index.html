WordPress.com vs WordPress.org

Believe it or not, they're different things!

WordPress.com

  • WordPress.com is "free"
    • Similar to Blogger, Tumblr, TypePad etc.
  • They host your files.
  • By default your domain name will be: YourName.WordPress.com
  • Customization options are extremely limited.
  • Your site will have ads, until you pay to remove them.
  • Your blog will be part of a built-in network.

Self Hosted : WordPress.org

  • Self-hosted WordPress sites run the WordPress software, which can be downloaded for free at WordPress.org
  • For the site to be online, you still must purchase a domain and hosting plan
  • The sky's the limit with your self-hosted WP site!

WordPress.com vs. a Self-Hosted Site

Which is right for you?
  • Self-hosted sites look more professional, can be fully customized, and have the potential for features like e-commerce and monetization
  • WordPress.com sites are easier to set up, managed for you, and expose you to a built-in network of bloggers

WordPress.com vs. a Self-Hosted Site

In this class we'll be focusing on self-hosted WordPress sites, but a lot of the features we'll use are available on WordPress.com as well.

Learn More: WordPress.org vs WordPress.com Infographic

Note: It's easy to move content from a WordPress.com site to a self-hosted site.

How Websites Work

What is a Website?

A website is a collection of folders (directories) and files (documents) that are accessible online.

What is a Website?

For our website to be accessible online, we need those folders and files to live on a computer that is always online.

These computers are called servers.

Web Hosting

When you buy a web hosting plan, you are buying space on a server for your website (folders + files) to live.

Domain Name

The domain name is the address at which people can find your website.

Example: http://grandcircus.co

Buying Hosting Plan + Domain Name

You can buy these separately, but typically you'll purchase them together as part of the same plan

There are TONS of hosting companies out there!

Read reviews of hosting companies, based on their performance with WordPress

Managed WordPress Hosting

  • Manages all technical aspects of WordPress for you
  • More expensive than typical hosting plans
  • Great for e-commerce and enterprise-level sites

Installing WordPress: Manual

Create a MySQL database on your hosting account

Installing WordPress: Manual

Edit 4 lines in wp-config-sample.php


				// ** MySQL settings - You can get this info from your web host ** //
				/** The name of the database for WordPress */
				define('DB_NAME', 'database_name_here');

				/** MySQL database username */
				define('DB_USER', 'username_here');

				/** MySQL database password */
				define('DB_PASSWORD', 'password_here');

				/** MySQL hostname */
				define('DB_HOST', 'localhost');

				/** Database Charset to use in creating database tables. */
				define('DB_CHARSET', 'utf8');

				/** The Database Collate type. Don't change this if in doubt. */
				define('DB_COLLATE', '');

Save the file and rename it to wp-config.php

Installing WordPress: Manual

Upload the WP software to your hosting via FTP

  • FTP, or file transfer protocol, is a tool for moving files between your computer and your web host
  • You'll need an FTP client to transfer files. There are lots of free ones, like Filezilla and Cyberduck
  • Your web hosting provider can help you get set up with FTP if you get stuck!

Getting Started with Your WordPress Website

The Dashboard

  • What you'll see when you log in to your WordPress site
  • Find it at yoursite.com/wp-admin

Configure Your Basic Settings

The first thing to do with any new install of WordPress is to make sure we get off to a good start with the correct settings!

This cheat sheet from WPUniversity is a great resource for remembering which settings are generally best.

Users > Your Profile

  • Personal Options: Set preferences to customize your WordPress Dashboard
  • Name: Choose how your name will be publicly displayed.
    • Recommended: Keep your public display name different from your username
  • Contact Info: Keep this up to date!
  • About Yourself: This is where you change your password!

Settings > General

  • Make sure to change that tagline!
  • Keep the email address here up to date always. It's what WordPress will use to contact you about any changes made to your site.
  • This is also where you can configure things like the date format and time zone – double-check that they're right!

Settings > Reading

  • Choose which page displays as your homepage! (We'll come back to this once we have some pages and posts in place.)
  • Choose how your blog posts display.
  • Search engine visibility: can be useful if you're not ready to have your site fully accessible to the world yet. Don't forget to uncheck it when you do launch!

Settings > Permalinks

  • Permalinks are permanent, static hyperlinks to a particular web page or entry in a blog

Settings > Permalinks

  • Default permalinks are ugly: http://yoursite.com/?p=123
  • Post names work well for most sites: http://yoursite.com/contact-us/
  • Some combination of date & name works well for sites with lots of timely content (news sites or blogs, for example):
    • Day and name: http://yoursite.com/2017/04/16/sample-post/
    • Month and name: http://yoursite.com/2017/04/sample-post/
  • You can even set up a custom structure

Pages vs. Posts

To use a page or a post? That is the question!

Pages vs. Posts

Pages typically serve as the backbone of your site.

Though you can edit them, they're generally static, standard info.

Common pages:

  • About
  • Contact
  • Team

Pages vs. Posts

Posts can be used lots of ways depending on the kind of site you're building, but are generally for more time-sensitive updates.

  • "Blog posts" is where we get the word "posts"
  • An artist might use "posts" as portfolio entries
  • A company could use "posts" to share news

Pages vs. Posts

Pages are timeless; posts are timely.

Tips for Creating Good Content

Menus

Navigation Menus

Menus allow visitors to find the major pages on a website

Primary navigation menus are often found in the header of the site, while additional secondary navigation menus may appear in sidebars, footers, etc.

Create a Menu

To create menus for our site, we navigate from the Dashboard to Appearance > Menus

Create a Menu

  • Your menu needs a name. Only site administrators will ever see this, but it's a good idea to name it something logical like "Main Menu" or "Services."
  • Add pages, posts, categories, or links from the left-hand column to the menu.

Create a Menu

You can drag & drop the menu items to reorder them, or even create sub-items (will display as dropdowns in most themes)

Create a Menu

Click the arrow on the right-hand side of the menu item to expand it and see further options.

WordPress Trick: Check the screen options for more fields

Where can we put menus?

  • This is another thing that depends on the theme
  • Themes come with menu areas, which are the places menus can go
  • Menus can also live in widgets, which we'll learn about shortly

Link Your Content

Another way to move visitors through your site is by linking your content effectively.

  • Use categories and tags on your posts
  • Consider a "related posts" plugin
  • When you mention something you've written about before, link back to it!

What is a Widget?

  • Widgets are content areas that can be added to your theme's widget areas, which often include sidebars, footers, and headers
  • Widgets often pull in dynamic content from your site or your social media feeds

What is a Widget?

Add Widgets to Your Site!

To manage widgets, we navigate to Appearance > Widgets in the Dashboard

  • The left-hand column shows our available widgets
  • The right-hand column shows our theme's widget areas
  • Drag & drop widgets from left to right

Editing Widgets

  • Note that there is no big blue "Save" button on our widgets screen

  • If you do edit the widget content, you'll need to save that
  • Not sure you want a widget on your site, but not ready to scrap it completely? Move it to the "inactive widgets" area in the lower left column

Extending WordPress with Plugins

Plugins

Plugins are add-ons for WordPress

  • Some plugins add content our users will see
  • Some plugins help us manage our site better
  • Plugins can be free or premium ($$)
  • Plugins != Widgets

What do plugins look like?

  • Plugins are directories containing code (always some PHP, often some JavaScript and CSS as well)
  • When we add a plugin to our site, WordPress saves it in wp-content/plugins

Manage Plugins

To manage the plugins on your site, navigate to Plugins in your Dashboard

Where do you find new plugins?

  • WordPress maintains a plugin directory
  • Access the directory from your Dashboard under Plugins > Add New
  • Access the directory online at wordpress.org/plugins

How to spot a good plugin

  • How recently has it been updated?
  • What do the reviews say?
  • Are the support threads resolved?
  • Is it compatible with the latest version of WordPress?

After you install a plugin...

  • Configure options
  • Does it work the way you expected?
  • Keep it up to date!

Customizing Your WordPress Site's Design

WordPress Themes

Themes modify the way your site is displayed, without modifying the underlying software.

In addition to changing the look of your site, many themes come with additional functionality.

Theme Features

  • Page Templates
  • Widget Areas
  • Menu Areas

Finding Themes

  • Like plugins, themes have a directory you can access in your Dashboard (Appearance > Themes > Add New) or online (wordpress.org/themes)
  • You can also find themes on the web at large, or make your own!
  • Themes can be free or premium.

Finding a Good Theme

When possible, view a demo of the theme in action. Ask:

  • Is it responsive?
  • Does it load quickly?
  • Is anything broken?
  • What do others say?
  • Has it been updated recently?
  • Does it have the content areas you need?

Security for WordPress

Is WordPress Secure?

Best Practices for Security

  • Keep WordPress up to date!
  • Keep plugins and themes up to date
  • Install themes & plugins from trusted sources only
  • Don't use "admin" as a username, and keep your username private when possible
  • Use strong passwords, and change them frequently
